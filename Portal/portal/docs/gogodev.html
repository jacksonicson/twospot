GoGo-Anwendung im DevServer

Nachfolgend werden die Schritte aufgeführt, um die GoGo-Anwendung im DevServer auszuführen.

Im ersten Schritt müssen der DevServer und die GoGo-Anwendung hier heruntergeladen werden.

Nach dem Download muss der DevServer in ein beliebiges Verzeichnis entpackt werden. Das gewählte Verzeichnis muss anschließend in die PATH-Umgebungsvariable aufgenommen werden. Unter Windows erfolgt dies über die System Properties, Environment Variables.... An die PATH-Variable der User variables oder system variables muss das Verzeichnis des DevServers angefügt werden. Befindet sich der DevServer im Verzeichnis C:/devserver muss an eine PATH-Variable ;C:/devserver angehängt werden.

Anschließend muss das Scratch-Verzeichnis im DevServer konfiguriert werden. Die entsprechende Konfigurationsdatei befindet sich im /conf/appserver/etc/-Verzeichnis und heißt appServer.properties. Der Schlüssel appServer.scratchdir muss auf ein existierendes Verzeichnis z.B. C:/temp/scratch zeigen.

Die Konfiguration des DevServers ist damit abgeschlossen. Nun kann die GoGo-Anwendung in ein beliebiges Verzeichnis extrahiert werden. Nach der Extraktion kann die GoGo-Anendung mit dem DevServer gestartet werden. Dazu wird der Befehl twospot.py --runServer im Verzeichnis der GoGo-Anwendung ausgeführt.

Nach dem Start des DevServers ist die Startseite der GoGo-Anwendung unter der http-Adresse http://localhsot:8080 erreichbar.
