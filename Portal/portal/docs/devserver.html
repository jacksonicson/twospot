DevServer

Setup

Das DevServer ZIP-Paket kann hier heruntergeladen werden. Das ZIP-Paket kann in ein beliebiges Verzeichnis extrahiert werden. Das gewählte Verzeichnis muss in die PATH-Umgebungsvariable aufgenommen werden. Damit wird der twospot.py-Befehl zur Verfügung gestellt.

Konfiguration

Vor dem Start des DevServers müssen eninige wenige Konfigurationseinstellungen vorgenommen werden. Diese betreffen das Scratch-Verzeichnis. Dieses wird vom DevServer für temporäre Dateien genutzt. Zum Beispiel werden JSP-Seiten in das Scratch-Verzecihnis kompiliert.

Die Konfigurationseinstellungen befinden sich in der devserver/conf/appserver/etc/appServer.properties Datei. Der anzupassende Schlüssel ist appServer.scratchdir (Zeile 11). Das folgende Beispiel zeigt eine korrekt konfigurierte appServer.properties-Datei.

Das Scratch-Verzeichnis muss bereits existieren! Es wird nicht vom DevServer erstellt!

#### Server Options ###
# Mode in which the AppServer is running (DEVELOPMENT, SERVER)
appServer.mode=DEVELOPMENT

# Defines the working directory in which the extracted application directores reside
appserver.workDir=

# Defines the scratch dir wich is used by the JSP-Compiler (TODO: Currently unused!!!)
appServer.scratchdir=C:/temp/scratch

### Security options ###
# Request-Time which is detected as a DOS-attack (in ms)
appServer.security.DOSPrevention.time=40000

Start

Der Start des DevServers erfolgt über das twospot.py Tools-Script. Dieses muss in dem Verzeichnis der auszuführenden Anwendung gestartet werden. Im Beispiel befindet sich die Anwendung gogo im Verzeichnis /home/alf/dev/gogo. Der DevServer wird dann folgendermaßen gestartet: 

/home/alf/dev/gogo> twospot.py --runServer

Nach dem Start ist der DevServer auf dem Port 8080 verfügbar. Die gogo-Anwendung ist z.B. über die http-Adresse http://localhost:8080/ erreichbar.
