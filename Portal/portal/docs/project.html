Build

Die TowSpot-Quellcodes sind in einer Reihe kleinerer Eclipse-Projekte organisiert. Jedes Projekt enthält eine ANT-Buildfile mit dem Namen build.xml. Das Skript erstellt aus dem Projekt einer JAR-Datei und legt diese zentral im Libs-Projekt ab. Dort befinden sich auch die Abhängigkeiten zur 3Party-Libraries. Die Projekte hängen selbst nur von den JAR-Dateien im Libs-Projekt ab. Direkte Eclipse-Projektabhängigkeiten existieren nicht!

Die ANT-Buildfiles weisen folgende Voraussetzungen auf:

  • Python 2.6.4
  • Google ProtoBuf (Compiler) 2.2.0

Um die gesamten TowSpot-Plattform zu kompilieren befindet sich im Build-Projekt eine zentralen ANT-Buildfile mit dem Namen plattform.xml. Diese ruft die Buildfiles der einzelnen Projekte in der richtigen Reihenfolge auf. Anschließend erstellt sie das ZIP-Archiv twospot.zip, das die gesamte TowSpot-Plattform enthält. Im Build-Projekt befinden sich noch die portal.xml-Buildfile, die ein WAR-Archiv aus der Portal-Anwendung erstellt.

Installation

Voraussetzung für den Betrieb der TwoSpot-Plattform ist eine funktionsfähige Installation der HBase-Datenbank in der Version 0.20.2. Das von der plattform.xml-Buildfile erstelle ZIP-Paket enthält die gesamte TwoSpot-Plattform. Die Konfigurationseinstellungen werden in der info.txt-Datei im Build-Projekt genauer erläutert.
