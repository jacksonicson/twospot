Adresse twospot.informatik.fh-augsburg.de kann nicht gefunden werden

Es wird nicht der DNS-Server der FH verwendet. Bei Einsatz eines externen DNS-Server kann der Hostname sporadisch nicht aufgelöst werden. Abhilfe schafft die Eintragung des DNS-Servers der FH als alternativen DNS-Server. Dies funktioniert auch bei Rechnern außerhalb des FH-Netzwerkes. Die IP-Adresse des DNS-Servers ist: 141.82.16.242


Log4j-ERROR beim Starten vom DevServer

Beim Starten vom DevServer wird der folgende Fehler angezeigt: 

log4j:ERROR Could not connect to remote log4j server at [localhost]. We will try again later.

In der Log4j-Konfiguration wurde ein Socket-Appender konfiguriert. Über diesen können die Log4j-Meldungen über einen Socket an einen Socket-Server (Log-Server) geschickt werden. Für den Betrieb des DevServers wird kein Log-Server benötigt und die Meldung kann ignoriert werden.

Für eine übersichtliche Darstellung der Log-Meldungen kann z.B. der Log-Server Lumbermill eingesetzt werden.


Das Starten einer Anwendung dauert sehr lange

Beim ersten Start einer Anwendung wird exklusiv ein neuer AppServer-Prozess gestartet. Nachdem der AppServer-Prozess gestartet wurde, werden alle Requests mit "normaler" Geschwindigkeit verarbeitet. Das Starten des AppServer-Prozesses dauert leider noch etwas länger, da die AppServer-Prozesse noch nicht im Cache gehalten werden.

Die Startgeschwindigkeit ist zudem abhängig von der Anwendung selbst. Führt die Anwendung bei der Initialisierung sehr viel Code aus, so dauert der Start auch dementsprechend länger! Die Startgeschwindigkeit einer einfachen HelloWorld-Anwendung beträgt zwischen 7 und 15 Sekunden.
